## 🔧 现在的测试步骤：

### 1. 配置API密钥（二选一）：

**方法A：PDF界面直接配置（推荐）**
1. 打开应用，点击"PDF演讲稿"标签
2. 在"API密钥"输入框输入您的OpenAI API密钥
3. 点击"保存密钥"按钮
4. 看到"API密钥已保存！"提示

**方法B：设置界面配置**
1. 点击左下角"设置"
2. 找到"LLM大模型"部分
3. 确保选择"OpenAI"服务
4. 在"API Key"字段输入密钥

### 2. 测试PDF处理：

1. **上传PDF文件**：
   - 点击文件夹图标选择PDF
   - 或直接拖拽PDF到界面

2. **配置生成选项**：
   - 选择AI服务：OpenAI GPT
   - 可自定义提示词（可选）
   - 勾选要生成的内容：演讲稿、字幕、TTS语音

3. **开始生成**：
   - 点击"开始生成"按钮
   - 观察进度条和状态信息
   - 在预览区查看生成的演讲稿

4. **查看结果**：
   - 看到"处理完成！生成文件：..."提示
   - 点击"打开文件夹"查看生成的文件
   - 或点击"保存结果"复制到其他位置

## 📁 生成的文件：

处理完成后会生成：
- `PDF名称_演讲稿.txt` - 智能生成的演讲稿
- `PDF名称_字幕.srt` - 带时间轴的字幕文件
- `PDF名称_语音.wav` - Mac TTS生成的语音

## 🔍 故障排除：

### 如果API调用失败：
1. 检查API密钥格式（必须以sk-开头）
2. 确认网络连接正常
3. 验证OpenAI账户有余额
4. 查看应用日志窗口获取详细错误

### 如果没有生成文件：
1. 点击"打开文件夹"按钮查看工作目录
2. 检查PDF文件是否能正常读取
3. 确认所有生成选项都已勾选

### 如果TTS失败：
- Mac系统会自动使用系统语音
- 从日志可以看到使用的语音引擎
- 语音文件会保存为WAV格式
